## 一句话判断

PromptsRef 和 OpenNana 要和 MeiGen 放在同一层级观察：它们都不是泛 AI 工具站，而是把 prompt、结果图、模型标签、分类/榜单和生成入口组合成可复用内容资产的 prompt gallery 竞对。

对 VogueAI 最重要的不是复制它们的 prompt，而是观察三件事：

```text
创作者账号如何持续供给视觉 prompt
-> 站内如何把 prompt 做成可浏览、可搜索、可生成的资产
-> 外部社媒、SEO、榜单、指南和生成器如何共同拉动回访
```

## 和 MeiGen 的差异

| 维度 | MeiGen | PromptsRef | OpenNana | VogueAI 要吸收什么 |
| --- | --- | --- | --- | --- |
| 核心内容源 | X viral prompt + creator credit | Underwood 自有整理 + SREF / prompt / guide | 松果先森和多来源中文 prompt 案例 | 既看 X 爆款，也看创作者自建库和中文本地 prompt 供给。 |
| 页面形态 | Gallery + prompt detail + generator + MCP/docs | Prompt library + ranking + guides + multi-model generator | Prompt gallery + model/media/tag filters + generate entry | 首页、详情页、榜单、分类页和生成器要串起来，不要只做静态列表。 |
| 增长入口 | X creator seeding、GitHub/MCP、SEO | X 创作者账号、榜单、指南、SREF 搜索、工具入口 | 中文社媒、提示词合集、站内大量 prompt 页 | VogueAI 需要同时做英文 creator、中文本地素材和 SEO/Pinterest 承接。 |
| 可借鉴重点 | creator credit 和 one-click generate | SREF/风格榜单、指南层、模型工具入口 | 中文标签体系、大量 prompt detail、收藏/生成路径 | 把 prompt asset 做成可复用、可生成、可分发、可解释的资产。 |

## VogueAI 立即动作

1. 把 `@underwoodxie96` 保留为 A0 视觉 prompt 账号，并标记为 PromptsRef 创作者，不只是普通素材号。
2. 把 `@songguoxiansen` 加入 Prompt / 视觉案例竞对池，按 OpenNana 创作者追踪。
3. 每次做 prompt asset 候选时，把 MeiGen、PromptsRef、OpenNana 作为同级竞对信号源。
4. 做竞对 QA 时固定检查：prompt 数量、模型筛选、标签分类、榜单/排序、source credit、生成 CTA、详情页 SEO、sitemap、schema、pricing、收藏/历史记录。
5. VogueAI 的对外口径不能只说“prompt gallery”，要强调“structured prompt asset + source credit + Use Prompt / Use as Ref + multi-model workspace”。

## 风险边界

- 不能直接搬运 PromptsRef / OpenNana 的 prompt 和原图；只能做 inspired-by、改写、变量化和自生成媒体。
- 对外发布时必须保留来源记录；如果是从竞对站看到线索，应优先追原始 X/社媒来源。
- Underwood 和松果先森既是创作者也是产品对手，互动/合作要和内容复用边界分开处理。
